The pen-and-paper flow, made virtual: free text on one connected rail of drawn arrows. One ink, one rail color, weight for hierarchy. Boxes appear only when a step needs extra clarification.

What you write

Verbatim from the doc — a root line, -> steps, indentation for sublayers. Backticks mark code values. A line starting with > is a clarification note, not a step.

Component

The waterfall

Every step hangs off the rail with a real arrowhead — the drawn version of the ->. Scope heads carry weight, deeper steps recede slightly, loop words read as keywords. Notes sit in the only boxes on the page, deliberately off the rail.

Deliberately absent: kind colors, icons, glyph markers, collapse affordances. Structure comes from the rail, hierarchy from weight, emphasis from bold keywords and code chips. The arrow-tree text is the single source; notes are the > lines.
