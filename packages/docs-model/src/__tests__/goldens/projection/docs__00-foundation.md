The foundation layer holds the system's durable intent. It records why documentation transfers intent and preserves trust, then turns that purpose into the principles that shape the shared artifact, block vocabulary, stable identity, customization boundary, and visual restraint. A reader descends only when the task needs the underlying argument.

## The Intent Register

- Manifesto

  - Why documentation exists: knowledge transfer, trust, and preserving human intent as agents act on code.

- Design Principles

  - Why the system uses one addressable artifact, equal reading surfaces, closed blocks, bounded customization, and restrained visuals.
